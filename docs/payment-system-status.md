## 技术架构

### 支付流程
```
1. 用户确认订单 → 创建订单 → 跳转支付页面
2. 支付页面 → 调用支付API → 获取微信支付参数
3. 支付页面 → uni.requestPayment() → 发起小程序支付
4. 微信服务器 → 支付结果回调 → 后端更新状态
5. 前端轮询 → 获取支付结果 → 显示支付状态
```

### 安全特性
- **防重复支付**：检查10分钟内同一订单的成功支付记录
- **支付签名验证**：模拟实现，可替换为真实微信支付签名验证
- **订单归属验证**：确保用户只能操作自己的订单
- **支付日志**：记录所有支付操作，支持审计
- **支付超时**：30分钟未支付自动标记为过期

### 错误处理
- **网络错误**：自动重试机制
- **支付失败**：清晰错误提示
- **超时处理**：自动停止轮询并提示
- **状态同步**：轮询确保前后端状态一致
